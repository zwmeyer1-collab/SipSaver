# SipSaver vs AppyHour

## Positioning

AppyHour appears to be strongest as a polished promotions and discovery platform with a business-side publishing motion for bars, restaurants, suppliers, and distributors.

SipSaver should win by being:

- more local
- more current
- more trustworthy
- more fun to use on a night out

That means we do not try to beat AppyHour first on partnerships or ad-tech. We beat them first on freshness, transparency, and Tampa-native utility.

## Product Thesis

AppyHour helps people discover structured specials.

SipSaver should help people answer:

- What is actually good near me right now?
- Which deals are verified today?
- Which places are worth hopping between tonight?
- What changed since the last time I checked?

## Competitive Matrix

| Area | AppyHour | SipSaver advantage |
| --- | --- | --- |
| Core content | Happy hours, specials, events, promos | Happy hours, specials, events, plus beer-forward "Hoppy Hour" and neighborhood crawl utility |
| Data model | Likely partner-entered / managed listings | Hybrid: places APIs + scraping + review queue + later venue claims |
| Freshness | Structured, but opaque on sourcing | Visible freshness timestamps, trust score, source links, update history |
| Venue coverage | Strong where they have structure/partners | Better long-tail local coverage from scraping websites + Instagram-first leads |
| UX | Polished browse/map/feed experience | More action-oriented: "best right now", "crawl this area", "verified today", "cheap drinks nearby" |
| Map | Discovery map | Discovery map plus route/crawl planning and clustered nightlife mode |
| Events | Included | Included, but tied to nightlife intent and neighborhood energy |
| Operator workflow | Business hub, profile claims, analytics | Start with internal ops queue, then add claim flow after demand exists |
| Notifications | Proximity and push | Freshness alerts, starting soon alerts, watchlist alerts, deal changed alerts |
| Brand activations | Strong | Not a v1 priority |
| Tampa fit | Multi-city product | Tampa-first, block-by-block launch playbook |

## Where SipSaver Should Beat Them First

### 1. Trust and freshness

Every listing should show:

- source type
- last checked time
- confidence
- whether it was scraped, imported, or manually approved

If we become the app users trust more, we win even with fewer listings.

### 2. Better nightlife decision-making

Instead of only showing listings, we should help users decide:

- best deals near me right now
- best places for beer, cocktails, tacos, trivia, brunch
- where to start in SoHo, Downtown, Ybor
- where to go next after one stop

This makes SipSaver feel more like a night-out assistant than a static directory.

### 3. Tampa-native density

We should own three launch zones before worrying about broad coverage:

- SoHo
- Downtown Tampa / Water Street / Channelside
- Ybor

If those zones feel alive, the app feels alive.

### 4. Hoppy Hour

This should become a real feature pillar:

- breweries
- taprooms
- draft specials
- flights
- beer releases
- tap takeovers
- dog-friendly patios
- game day beer spots

This is a memorable hook that separates SipSaver from generic happy-hour products.

## What We Should Build

## V1: Beat them on usefulness

- real map with nearby venues
- search + neighborhood filters
- live review queue
- real place imports
- venue-to-place matching
- prospect scraping for new places
- imported deal snippets from venue websites
- freshness + confidence display on every deal

Success signal:

Users can open the app in Tampa and reliably find a few real, current options within seconds.

## V2: Beat them on delight

- "Best right now" feed
- "Starting soon" feed
- crawl mode for SoHo / Ybor / Downtown
- save favorite spots
- watchlist alerts
- Hoppy Hour tab
- better venue pages with hours, photos, tags, menu links, directions

Success signal:

The app becomes something users open before and during nights out, not just when they are randomly searching.

## V3: Beat them on scale

- claim your bar flow
- bar owner dashboard
- self-serve specials publishing
- sponsored placements
- supplier / distributor activation tools
- analytics for bars

Success signal:

The product starts creating its own structured data instead of relying mostly on scraping and ops review.

## Best Near-Term Feature Priorities

If we want the highest leverage next, build these in order:

1. Promote scraped prospect deals into real consumer-facing cards.
2. Add freshness/trust labels to the deal feed and venue views.
3. Build venue detail pages.
4. Add "Best right now" and "Starting soon" sort modes.
5. Add Hoppy Hour as a dedicated filter or tab.
6. Add save/watchlist behavior.
7. Add crawl mode for Tampa neighborhoods.

## Data Strategy

Short term:

- Google Places or Mapbox for venue discovery
- scrape official websites for deals
- track imported snippets in a review queue
- manually approve top-value items

Medium term:

- Instagram ingestion / lead detection
- recurring change detection
- auto-archive stale deals
- venue pages that combine place data with verified offer data

Long term:

- owner-managed dashboards
- business claims
- partner feeds
- sponsored activations

## Brand Strategy

AppyHour sounds broad and established.

SipSaver should feel:

- brighter
- more local
- more energetic
- more beer-and-nightlife fluent
- more transparent

The message should be:

"Find the deals that are actually worth going to tonight."

## One-Sentence Strategy

SipSaver wins by becoming the most trusted, most current, and most Tampa-native way to decide where to drink, eat, and hop next.
